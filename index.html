Project: stayTuned


Preview Source Code

The minimal viable product for this project is to provide the User with information about a trending artist from the stayTuned platform. If a person is interested in a particular artist, they are able to discover the artist's top songs, its lyrics and any upcoming tours.


  • HTML
  • CSS
  • JavaScript
  • jQuery
  • Firebase
My Role

In this project I helped my group connecting to the Lyrics.ovh API and building the Firebase storage of User information. Not only would Firebase store user inputs of searched artists, but I worked to get the most searched musicians to populate on the website's index page.


Project Challenges

There was a learning curve to manipulate and modify data on Firebase! In the beginning it was focused on preventing the database from storing new data for artists already searched, then directing searches to its proper objects.


My Solution

Firebase stores the new searches under a key name no matter what node you direct the search entries onto. The solution to this challenge was to target the entry key upon each artist search, and attach it as an id for a specific musician.


Notable Features

My group and I managed to implement ajax calls to four different APIs. iTunes API for the music preview, BandsInTown to populate when the artist is on tour, Lyrics.ovh to get the lyrics from the top three songs and a Wikipedia API for a short bio on the musician.
